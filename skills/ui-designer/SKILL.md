# UI Designer

Use this for design-system extraction and matching, not for ordinary site critique.

## Inputs

Screenshots, URLs, local pages, Figma context, Open Design artifacts, existing code, tokens, component libraries, or brand references.

## Extract

- visual language: typography, color, spacing, radius, elevation, density, imagery;
- component inventory: buttons, inputs, cards, tables, navigation, modals, alerts, forms;
- states: hover, focus, active, selected, disabled, loading, empty, error;
- layout rules: grid, breakpoints, container widths, rhythm, section patterns;
- accessibility notes: focus, contrast, labels, keyboard behavior, reduced motion;
- do and don't rules for future work.

## Output

Create or update a compact `DESIGN.md` when useful. Include tokens, components, states, accessibility, and examples. Keep it practical enough for implementation.

## Routing

- Use this for extraction or matching.
- Use `site-design-audit` for critique of an existing site.
- Use `site-redesign` for changing an existing site.
