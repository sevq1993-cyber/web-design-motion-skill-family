# Site Design Create

Use this to create a site or page from a brief.

## Start With

- page type and goal;
- target audience and user need;
- offer, product, or object being shown;
- required sections and real content;
- brand tone and visual references if available;
- stack and project constraints.

## Build The Experience

For landing pages, make the product, offer, or category clear in the first viewport. Show a hint of the next section on common desktop and mobile viewports.

For dashboards and product tools, skip marketing structure. Build the useful working screen first: navigation, data density, actions, states, filters, forms, and feedback.

For portfolios or editorial pages, make the object or person visible early and use media intentionally.

## Shared Gates

Use `frontend-design` for visual direction and `frontend-ui-engineering` for implementation quality. Use `frontend-motion-style-system` only when animation, scroll, media, or 3D is part of the requested experience.
